# 资源库管理、Tag 分类、设置与交互设计

> 状态：讨论结论稿  
> 日期：2026-07-19  
> 适用平台：Windows / macOS  
> 关联文档：[图片系列、对比实验与生成分支设计](./experiment-and-branch-design.md)

## 1. 目标

当前作品库和 Vibe 库已经能够导入、查看和使用资源，但还缺少完整的管理能力；Tag 分类、AI 配置和界面偏好也分散在局部界面中。本文定义四项相互关联的能力：

1. 作品与 Vibe 的人工整理、批量操作和安全删除。
2. 可扩展的 Tag 分类体系以及分类、极性、语法诊断之间的边界。
3. 独立的软件设置页面和可版本化的 AI 任务配置。
4. 统一、可访问、跨平台的拖拽与动效规则。

本文不替代实验与分支模型。资源库回答“东西放在哪里、如何找到和管理”，实验与分支回答“图片之间存在什么生成关系”。

## 2. 已确认的产品原则

1. 一张作品或一个 Vibe 可以同时属于多个收藏集。
2. 拖入收藏集表示增加成员关系，不移动实体文件。
3. 从收藏集移除、移入回收站和彻底删除是三种不同操作。
4. 用户可以为未知 Information Extracted 设置具体数值，但必须标记为“用户设置，未验证”。
5. 用户设置数值不会重新计算或修改 `.naiv4vibe` 文件。
6. AI Prompt 默认使用受保护的安全模板；完整覆盖只在高级模式中开放。
7. 每一种拖拽操作都必须同时提供按钮、菜单或键盘操作。
8. Windows 与 macOS 使用同一数据语义，但菜单、快捷键、路径、窗口和安全存储遵循各自平台习惯。

## 3. 概念边界

| 概念 | 用途 | 是否表达生成关系 | 是否允许多重归属 |
| --- | --- | --- | --- |
| 收藏集 Collection | 人工整理作品或 Vibe | 否 | 是 |
| 智能筛选 Smart Filter | 按条件临时或保存筛选 | 否 | 是 |
| 系列 Series | 宽松描述一组相关创作结果 | 是，但不声称因果 | 是 |
| 对比实验 Experiment | 固定和比较生成变量 | 是 | 是 |
| 分支方案 Branch Recipe | 从结果或方案派生的新生成配置 | 是 | 有明确来源 |
| Tag 分类 Taxonomy | 描述 Prompt Tag 的语义 | 否 | 每个 Tag 一个主分类，可附加属性 |

“收藏集”和“系列”不能合并。收藏集可以包含“本周待整理”“喜欢的画风”这类纯管理分组；系列则属于创作关系模型。

## 4. 作品库管理

### 4.1 导航结构

作品库侧栏建议包含：

- 全部作品
- 未分组
- 收藏
- 回收站
- 用户收藏集
- 后续可加入的智能筛选

收藏集支持创建、重命名、排序、设置颜色、归档和删除。删除收藏集只删除成员关系，不删除作品。

### 4.2 成员管理

作品支持单选、多选和全选当前筛选结果。批量操作包括：

- 加入一个或多个收藏集
- 从当前收藏集移除
- 收藏或取消收藏
- 移入回收站
- 恢复
- 查看所属收藏集、系列和实验

同一作品可以属于多个收藏集。拖入收藏集的默认语义是“加入”；如果已经属于该收藏集，则提供安静的已存在反馈，不重复创建关系。

### 4.3 删除与回收站

删除采用两阶段流程：

1. **移入回收站**：作品在正常浏览和选择器中隐藏，但引用关系和应用管理的文件仍保留。
2. **彻底删除**：删除应用管理的图片副本、缩略图和数据库记录。

彻底删除前必须显示引用检查：

- 所属系列和实验数量
- 作为哪个分支的结果或来源
- 是否作为 Vibe 来源图
- 是否存在待生成方案引用

默认不删除用户导入位置的原始文件。应用只能清理自己管理目录中的副本和缩略图。存在引用时应优先引导恢复、替换引用或取消操作，不能静默级联破坏实验关系。

## 5. Vibe 库管理

### 5.1 组织方式

Vibe 库复用收藏集、多选、收藏、归档和回收站能力，但作品收藏集与 Vibe 收藏集相互独立。不要让用户在同一个收藏集中混合两种资源。

Vibe 专用筛选包括：

- 有来源 PNG / 来源缺失
- 已验证 / 用户设置 / 未知 Information Extracted
- 已编码 / 仅来源图片
- 模型兼容性
- 是否正在被作品、分支或实验引用
- 同一来源 PNG 的编码变体数量

同一 PNG 派生的多个 `.naiv4vibe` 继续以来源组展示；收藏集作用于 Vibe 资源，来源组表达派生关系，两者不能互相替代。

### 5.2 Information Extracted 状态

Information Extracted 使用三个来源状态：

| 状态 | 含义 | 界面表现 |
| --- | --- | --- |
| 已验证 Verified | 来自明确 metadata、原始 Vibe 信息或可靠映射 | 可用标记和已验证说明 |
| 用户设置 User-set | 用户声明现有编码对应某个数值 | 可用标记和“未验证”警告 |
| 未知 Unknown | 无法确定现有文件的计算位置 | 不显示已知可用位置 |

用户可以用 `0.00–1.00`、步长 `0.01` 的滑杆或数值框设置未知值。例如把一个现有文件设置为 `0.70` 后，只在 `0.70` 位置把该文件视为可用；移动到没有对应编码文件的位置时，仍显示 `.naiv4vibe` 不可用。

手动设置必须绑定到具体编码文件的指纹，而不是只绑定到显示名称或来源 PNG。文件被替换后需要重新确认，避免把旧声明错误套用到新编码。

### 5.3 删除规则

删除 Vibe 前显示引用数量和引用位置。存在引用时优先使用“归档”：已有作品和实验仍能解析它，但新建方案的选择器默认不再显示。彻底删除需要先解除或替换引用。

## 6. 拖拽与动效系统

### 6.1 技术职责

建议采用：

- `@atlaskit/pragmatic-drag-and-drop`：负责拖拽状态、放置目标、跨列表、外部文件和自定义预览。
- `motion`：负责列表重排、筛选、进入退出和选中状态的视觉连续性。
- CSS：负责简单的悬停、按下、焦点和颜色过渡。

拖拽库负责行为，动效库不接管数据操作；所有添加、移动、排序和删除命令必须先具备独立可测试的业务操作，再由拖拽调用。

### 6.2 视觉反馈

- 拿起时轻微上浮，缩放不超过 `1.02`。
- 原位置降低透明度但保留布局空间。
- 收藏集目标显示明确高亮；排序目标显示插入线或占位间隙。
- 多选拖拽使用简化叠放预览，并显示数量。
- 大图预览使用缩略图、名称和最多三项关键信息，避免 Windows 原生预览过大变暗。
- 无效目标显示拒绝状态，不允许“放下后才报错”。
- 支持边缘自动滚动、取消回位和成功后的短暂确认。
- 常规反馈控制在 `120–220ms`，使用自然减速，不使用弹跳或装饰性循环动画。
- 遵循系统减少动态效果设置，并允许在软件设置中进一步关闭动效。

### 6.3 可访问替代

拖拽永远不是唯一操作方式。每个拖拽结果都要有对应菜单命令，例如“加入收藏集”“移动到第 3 位”“移出当前收藏集”。键盘焦点、选中状态和操作结果不能只依靠颜色表达。

## 7. Tag 分类体系

### 7.1 建议的一级分类

1. 画师 Artist
2. 角色与主体 Subject / Character
3. 服装与饰品 Clothing / Accessory
4. 动作与姿势 Action / Pose
5. 场景与环境 Scene / Environment
6. 构图与镜头 Composition / Camera
7. 光照与色彩 Lighting / Color
8. 风格与媒介 Style / Medium
9. 质量与审美 Quality / Aesthetic
10. 文字与排版 Text / Typography
11. 内容与分级 Content / Rating
12. 未分类 Unsorted

`year2025` 建议归入“质量与审美 → 年代/审美时期”。它会影响模型的生成倾向，但与 `best quality` 等直接质量描述不同，因此需要二级语义而不是继续增加大量平级分类。

### 7.2 独立维度

以下信息不能作为普通分类混入上述体系：

- 正向 Prompt / Undesired Content：属于极性与所在 Prompt 区域。
- Base / Character：属于 Prompt 结构和作用域。
- `::`、括号不匹配和异常权重：属于语法诊断。
- AI 分类置信度：属于分类结果来源。

内置分类使用稳定 ID。用户可以调整显示名称、顺序、颜色和可见性，但不能删除稳定 ID；用户可以另外创建自定义分类。

### 7.3 AI 分类缓存

翻译和分类结果可以复用，但缓存键必须包含：

- 规范化后的原 Tag
- AI 服务商和模型
- 任务类型
- 分类体系版本
- Prompt 模板版本
- 输出语言

用户修改 Tag、分类体系或 AI Prompt 后，只失效受影响的结果，不能清空全部缓存，也不能继续误用旧分类。

## 8. 独立设置页面

设置采用完整页面，不放在检查器窄栏或多层弹窗中。页面保持左侧分区导航和右侧单列内容，避免设置卡片层层嵌套。

### 8.1 外观

- 跟随系统 / 浅色 / 深色
- 字体大小和界面密度
- 完整 / 减少 / 关闭动效
- 系统缩放与当前有效显示说明

明暗主题必须基于语义颜色变量，不应简单反转现有颜色。标题栏、菜单和系统控件要在 Windows 与 macOS 分别验证。

### 8.2 作品库与文件

- 应用管理目录和占用空间
- 导入时复制策略
- 重复文件处理
- 回收站保留时间
- 缩略图重新生成
- 自动归组建议开关

### 8.3 AI 服务

- 服务商、接口地址和模型
- API Key 安全存储
- 超时、并发和连接测试
- 翻译与分类可分别选择模型

密钥不写入普通数据库、日志、导出文件或跨平台协调文档，继续使用操作系统安全存储。

### 8.4 AI 任务

翻译和分类分别配置：

- 任务说明
- 输入变量预览
- 锁定的输出格式
- 示例输入与测试结果
- 恢复默认值
- 配置版本历史
- 导入和导出任务配置

普通模式只允许修改任务说明和少量参数；高级模式允许完整覆盖 Prompt，但必须先警告可能破坏返回格式，并在保存前执行结构校验。

### 8.5 Tag 分类

- 分类顺序、颜色、显示名称和可见性
- 自定义分类
- 二级语义和规则
- AI 分类预览
- 恢复默认分类体系

### 8.6 数据、备份与快捷键

- 数据库备份和恢复
- 缓存占用、按任务清理缓存
- 配置导入导出
- 快捷键查看和冲突提示
- 诊断信息导出，但默认移除密钥和私人路径

## 9. 与实验和分支设计的依赖关系

```text
稳定资源身份、收藏集、回收站、引用检查
    ├── 作品结果不可变与用户注释分离
    │       └── 分支方案、结果匹配、系列与对比实验
    ├── Vibe 来源组、编码指纹与 Information 来源状态
    │       └── 分支中可靠引用 Vibe
    └── 批量选择和统一命令
            └── 拖拽、动效和实验胶片排序

可版本化设置、Tag 分类和 AI 任务配置
    └── 正确缓存翻译与分类结果
            └── 智能收藏、自动归组和实验候选建议
```

如果先实现实验界面，再重构作品身份、删除规则和引用完整性，实验数据将需要二次迁移。相反，先只做主题或拖拽虽然可见度高，但不能解决资源关系和数据安全问题。

## 10. 推荐实施顺序

### 阶段 1：作品资产管理基础（建议从这里开始）

- 建立可版本化数据库迁移机制。
- 保持作品稳定 ID，区分用户原始路径与应用管理副本。
- 增加作品收藏集、成员关系、收藏状态和回收站状态。
- 建立引用检查服务和彻底删除规则。
- 完成多选及按钮/菜单批量操作。
- 在业务操作稳定后，为加入收藏集和排序接入拖拽预览。

这一阶段能立即解决“作品库无法管理”，同时为实验、分支和安全删除提供共同底座，范围也比直接重写实验系统更容易验证。

### 阶段 2：不可变结果与可编辑注释

- 冻结图片对应的生成事实。
- 将名称、翻译、分类、备注、收藏和成员关系定义为可编辑注释。
- 停止把 Prompt 修改直接写回原结果。
- 为现有 Prompt 版本准备兼容迁移，而不是立即删除历史数据。

### 阶段 3：Vibe 管理与来源可信度

- 为 Vibe 增加收藏集、归档、回收站和引用检查。
- 实现 Verified / User-set / Unknown 状态。
- 将用户声明绑定到编码指纹。
- 完成来源组的合并、拆分和缺失 PNG 提示。

### 阶段 4：设置、Tag 分类与缓存版本

- 建立设置页面骨架和安全存储边界。
- 扩展 Tag 分类并加入自定义分类。
- 建立 AI 任务模板版本和缓存命名空间。
- 再实现明暗主题、字号和动效设置，避免主题样式散落。

### 阶段 5：分支、系列与对比实验

- 引入分支草稿和来源关系。
- 将旧版本胶片迁移为实验/系列胶片。
- 实现结果上传与 metadata 匹配。
- 增加控制变量实验和参数差异视图。

### 阶段 6：高级智能与视觉完善

- 智能收藏和自动归组建议。
- AI 分类、翻译批处理和缓存管理。
- 多图同步对比和视觉相似度建议。
- 统一剩余拖拽、过渡、空状态和减少动态效果体验。

## 11. 第一阶段验收标准

第一阶段完成时，用户应当能够：

1. 创建、重命名、排序和删除作品收藏集。
2. 把一张作品加入多个收藏集。
3. 多选作品并批量加入、移出、收藏或放入回收站。
4. 从回收站恢复作品。
5. 在彻底删除前看到完整引用，并确认不会删除外部原图。
6. 使用菜单或键盘完成所有拖拽能完成的操作。
7. 在 Windows 与 macOS 上得到一致的数据结果和符合平台习惯的快捷键。

第一阶段不需要同时完成主题、完整设置页、AI Prompt 编辑器或实验分支界面。它的目标是先把资产生命周期和引用安全做正确。
