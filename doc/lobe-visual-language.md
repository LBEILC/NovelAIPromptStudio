# Lobe 视觉层级与颜色语义规范

## 目标

NovelAI Prompt Studio 使用 Lobe UI 的 `slate + blue` 主题，但应用的视觉气质不能只依赖组件外观。布局表面、按钮层级、状态色与 Tag 分类色必须遵循同一套规则，确保 Windows 与 macOS 上都保持安静、清晰、适合长时间工作的创作台体验。

本规范解决四类问题：

1. 大量深灰卡片造成界面被切成碎片；
2. 普通按钮使用蓝、绿、黄等彩色文字，操作层级不稳定；
3. Tag 的分类色、主题色、选中色和错误色彼此混用；
4. Lobe 组件默认样式被旧 CSS 再次覆盖，形成视觉漂移。

## 设计原则

- **连续表面优先**：工作区、导航区与检查器首先是连续平面，不为每个信息块创建卡片。
- **主色只表示交互重点**：蓝色用于主操作、当前选择、焦点与真正的链接。
- **语义色只表示状态**：绿色表示成功或已验证，黄色表示警告或待处理，红色表示错误或破坏性操作。
- **分类色与主题色分离**：Tag 分类使用独立色板，不能因为主题是蓝色就把 Tag 文字染蓝。
- **内容文字保持稳定**：普通内容、普通按钮与 Tag 主文字使用中性色，颜色只作为辅助线索。
- **不依赖颜色单独传达含义**：状态同时保留图标、文字或形状差异。

## 表面层级

### Level 0：工作画布

使用 Lobe `colorBgLayout`。用于图片舞台、Prompt 总览内容区、设置页背景等大面积区域。

### Level 1：结构面板

使用 Lobe `colorBgContainer`。用于左侧作品库、顶部工具栏、右侧检查器、版本胶片容器等固定结构。

### Level 2：轻交互表面

使用透明背景或极低比例的 `colorFill`。用于列表悬停、Tag、Vibe 条目、版本条目和批量工具栏。

默认不使用完整深灰卡片；优先通过间距、细分隔线和轻微悬停底色组织内容。

### Level 3：抬升表面

使用 Lobe `colorBgElevated`。仅用于 Context Menu、Popover、Modal、Toast、浮动匹配面板等真正位于其他内容之上的元素。

禁止在固定面板中连续嵌套 Level 2/3 表面。

## 按钮颜色语义

| 按钮类型 | 文字 | 背景/边框 | 使用场景 |
| --- | --- | --- | --- |
| Primary | 高对比度实色文字 | Primary 蓝色 | 导入、复制、保存、确认使用 |
| Default | 中性主文字 | Lobe 默认中性表面 | 普通操作 |
| Text / Ghost | 中性次级文字 | 透明；悬停使用中性填充 | 文件夹、设置、翻译、次级导航 |
| Danger | 红色或红色实底 | Lobe danger 变体 | 删除、废弃、移入回收站 |
| Status action | 仍按 Default/Text 处理 | 状态颜色只留在旁边的状态提示 | 打开已验证文件、恢复缓存等 |

普通按钮禁止因为所属内容“已编码”“安全”“AI”而使用绿色或蓝色文字。按钮颜色表达操作层级，不表达内容状态。

## 状态色

- **Success**：已保存、缓存命中、已验证、导入成功。
- **Warning**：缺少源图、参数未缓存、局部重绘不可复现、用户设置未验证。
- **Danger**：删除、不可恢复、metadata 不匹配、导入失败。
- **Info**：处理中、说明性信息。正文仍保持中性，只在图标或小型状态标记上使用蓝色。
- **Unknown**：使用中性次级文字，不得显示成成功绿色。

## Tag 分类配色

Tag 分类使用独立分类色板：

| 分类 | 色相角色 |
| --- | --- |
| 画师 | Magenta / Rose |
| 角色 | Cyan |
| 服装 | Purple |
| 场景 | Green |
| 风格 | Gold |
| 未分类 | Neutral |

具体颜色从当前 Lobe/Ant Design 主题色板取得，不写死为主题 Primary。

### Tag 组件规则

- Tag 主文字始终使用中性主文字色。
- 分类通过低饱和背景、完整边框和小圆点表达，不使用单侧色条。
- 权重数字可使用分类色的低饱和版本，但不能固定使用主题蓝。
- 选中状态使用 Primary 蓝色边框/底色和勾选标记，主文字仍保持中性。
- Undesired Content 使用低饱和危险色背景与边框，主文字保持可读的中性色。
- 语法异常使用黄色警告图标和虚线边框，不改变分类归属。
- 翻译文字使用次级中性色。

## 组件规范

- 优先保留 Lobe 组件的默认 `Default`、`Text`、`Primary`、`Danger` 视觉。
- 不通过业务类名把普通按钮的 `hover` 文字改成 Primary 或 Success。
- 业务 CSS 可以控制尺寸与布局，但不应重新实现组件的完整色彩状态。
- Context Menu、Alert、Toast、Segmented、Button、Checkbox、Input、Select 等继续使用现有 Lobe 组件。

## P1–P2 落地范围

### P1

- 拉平作品库、Prompt 检查器、Vibe 卡片、Vibe 库和版本胶片的固定表面。
- 普通按钮恢复中性色；Primary、Danger 和状态提示各司其职。
- 清理文件操作、设置、翻译、收藏等普通操作的彩色文字覆盖。

### P2

- Tag 编辑器和 Prompt 总览共用分类色变量。
- 选中状态不再把 Tag 内容文字强制染成主题蓝。
- 权重与分类提示不再固定走 Primary 色。
- 未知 Vibe Information 状态恢复中性色，已验证和用户设置分别使用 Success/Warning。
- 清理与上述规则冲突的旧 CSS。

## 验收清单

- 深色与浅色主题下，大面积固定区域不出现卡片套卡片。
- 普通按钮默认与悬停状态均为中性色。
- 页面同时出现的高饱和彩色操作不超过一个主操作和必要的状态提示。
- Tag 主文字不随主题 Primary 改色；切换主题色不改变分类色语义。
- Tag 选中、Undesired 和语法警告可同时识别，且不只依赖颜色。
- Vibe 的成功、警告与未知状态语义准确。
- Windows 与 macOS 使用同一规则；平台差异只保留菜单、窗口与快捷键习惯。
