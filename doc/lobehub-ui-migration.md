# Lobe UI 完整迁移试验

> 状态：主题接管与应用外壳第一批迁移完成
>
> 分支：`codex/lobe-ui-migration-spike`
>
> 日期：2026-07-19

## 结论

完整迁移在技术上可行，视觉方向也符合 NovelAI Prompt Studio 的定位。项目不再保留原有视觉规范，颜色、字体、圆角、基础控件、状态和动效统一以 Lobe UI 为准；现有的信息架构和专业工作流继续保留，不照搬 LobeChat 的聊天产品布局。

第一阶段已经将 Lobe UI 的主题环境、按钮、输入框、密码框和分段选择器接入设置页。Windows 实机验证表明，控件的视觉质量、间距和状态表达都优于此前的自制基础控件，可以继续迁移。

界面美观度与一致性优先于应用体积。完整迁移仍需记录以下技术代价，以避免它们演变成运行问题：

- `@lobehub/ui` 依赖 Ant Design、`antd-style`、Motion 等运行时，基础接入就会明显增加 JavaScript 体积。
- 当前版本存在 `@emoji-mart/react` 对 React 19 的 peer dependency 警告；虽不影响本次构建和运行，但升级时必须持续关注。
- 主题 Provider 会影响根节点尺寸、全局样式和浮层容器。本次实机验证已经发现并修复过一次根节点高度链断裂问题。
- 桌面应用无需承担网页首屏网络下载，但仍要关注启动解析时间、安装体积、打包时间和长期依赖维护成本。

## 已完成结果

### 已完成

- 安装并锁定 Lobe UI 5.20.3 及其必要 peer dependencies。
- 建立统一的 `LobeProvider`，默认使用 `slate + blue`，开启 Lobe 全局样式和中文区域设置。
- 支持深色、浅色与跟随系统三种主题模式，并将选择保存在设备偏好中。
- 将 Geist、Geist Mono 与 HarmonyOS Sans SC 作为本地资源随应用构建，不在运行时访问字体 CDN。
- 在设置页迁移按钮、输入框、密码框和分段选择器。
- 迁移左侧主导航、导入按钮、搜索框、顶部工作区切换、复制按钮、操作图标和多选框。
- 将旧语义颜色桥接到当前 Lobe Token，清理应用外壳和设置页的主要硬编码暗色，保证双主题对比度。
- 将 Lobe UI 及其递归依赖归入独立运行时构建块，便于缓存和分析。
- 在 Windows 桌面应用中验证主工作区、设置页、AI 设置页和原生窗口菜单状态。

## 迁移边界

完整迁移指的是：应用的基础主题、表单控件、菜单、弹层、提示、列表表面、卡片和状态反馈统一采用 Lobe UI 或基于其封装的项目组件。

以下部分继续保留项目自身设计：

- 作品库、中央画布、Prompt/Vibe 检查器组成的三栏工作台。
- Prompt 分类、多选、版本胶片和 Vibe 分组等专业信息架构。
- 图片与 ZIP 拖拽导入、文件系统操作、原生窗口、快捷键和安全存储。
- 需要高性能或复杂手势的拖拽排序与图片预览；Lobe UI 只负责其视觉表面。

## 分阶段顺序

### 阶段 A：基础设施与设置页（已通过）

- Provider、主题桥接和构建分组。
- Button、Input、Password、Segmented。
- Windows 实机验证和体积基线。

### 阶段 B：应用外壳与通用控件（第一批已完成）

- 已完成左侧导航、顶部工具栏、搜索、复选框、Select、作品批量操作工具栏和设置页导航。
- 下一批处理 Tooltip、Dropdown 和应用内弹层；操作系统原生右键菜单继续保留原生实现。
- 统一焦点态、禁用态、危险操作和键盘导航。
- 验证 Windows 与 macOS 字体、缩放、窗口宽度和浮层定位。

### 阶段 C：作品库与管理页面

- 作品列表、收藏集、回收站、手动分组和批量操作。
- Vibe 库、Vibe 与 PNG 绑定、缺失源图提示、打开所在文件夹。
- 空状态、加载态、错误态和导入进度。

### 阶段 D：Prompt 与 Vibe 工作区

- Prompt 总览的分类分组、多选、排序、翻译显示和复制流程。
- Base Prompt、Undesired Content、Character Prompt 与 Character Undesired Content 的层级展示。
- Vibe 的 Information Extracted 滑杆、已计算刻度和不可用状态。

### 阶段 E：版本分支与对比工作流

- 将源图片元数据保持只读，修改参数时创建待生成分支。
- 建立图片组、差异维度、分支状态和结果图片回填。
- 使用专门的拖拽方案实现排序、拖拽预览和跨组移动。

## 每阶段验收门槛

- 行为：现有功能与快捷键不回退，键盘和鼠标都可完成主要流程。
- 视觉：暗色/亮色、字体缩放和紧凑度设置一致生效。
- 性能：记录构建产物、启动时间、长列表滚动和图片切换响应。
- 平台：Windows 完成实机验证；涉及窗口、字体、文件和快捷键的变更必须交给 macOS 复核。
- 质量：通过测试与生产构建后才提交和推送；阶段完成后再决定是否合并到主线。
