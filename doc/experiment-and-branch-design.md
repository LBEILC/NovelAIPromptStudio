# 图片系列、对比实验与生成分支设计

> 状态：讨论稿  
> 日期：2026-07-19  
> 适用平台：Windows / macOS

> 2026-07-19 实施进度：分支基础、结果回传闭环、手动创作系列和严格对比实验的最小闭环已落地。原图生成事实由主进程强制保护；修改 Prompt、Vibe 或生成参数时会自动创建独立草稿；草稿可放弃或标记为待生成；历史 Prompt 版本可转为分支。待生成分支可上传 PNG，程序会核对 Prompt、Vibe、Seed、Model、Sampler、Steps 与 CFG 并保存匹配差异。不匹配时会保留预期方案、展示逐项差异，并按实际 metadata 自动创建结果子分支；一个方案可关联多个结果。创作系列与收藏集使用独立关系，一张结果图可以加入多个系列，删除系列不删除作品。多选结果可以建立以当前作品为基准的对比实验，成员变化后自动重算固定字段、变化字段以及“参数相同 / 单变量 / 混合变量”状态；缺失 metadata 不会被误报为固定条件。尺寸核对、成员拖拽排序、实验胶片和多图同步对比仍按本文后续阶段实施。

## 1. 背景

当前“版本胶片”将版本理解为同一图片下可保存和恢复的 Prompt 快照。但 NovelAI 生成图片中的 Prompt、Vibe、Seed、Model、Sampler、Steps、CFG、尺寸等参数，都是该图片已经发生的生成事实。把这些参数修改后继续绑定在原图上，会造成“图片与生成信息不再对应”的数据错误。

真实素材之间也不是线性的 V1 → V2 → V3 关系。一组图片可能具备相同 Seed、不同 Tag；相同 Base Prompt、不同 Character Prompt；相同 Prompt、不同 Vibe；或者同时变化多个参数。因此当前线性版本模型不能正确表达图片关系。

## 2. 真实素材观察

对 `E:\Works\AI\AIPictureNovelAI\风笛` 目录进行只读分析：

- 共 67 张 PNG，其中 55 张包含可用生成信息。
- 52 张包含 Character Prompt，24 张包含嵌入 Vibe。
- 存在“同 Base Prompt、不同 Character”“同 Prompt、不同 Vibe”“同 Prompt/Vibe、不同 Seed”三类重复模式。
- 最大同 Base Prompt 家族包含 15 张图片，同时出现 9 套 Character、15 个 Seed、5 套 Vibe。
- 12 张图片缺少足够 metadata，不能完全依赖生成参数自动归组。

结论：软件需要同时支持宽松归档、严格控制变量对比、来源分支和不完整 metadata，不能用一个“版本”概念覆盖全部关系。

## 3. 设计原则

1. 结果图是不可变事实，生成参数只能查看和复制。
2. 修改生成参数时创建新分支，不改变原结果图。
3. 用户注释与生成事实分离。翻译、分类、备注、评分、收藏和所属系列可以修改。
4. 图片归档与控制变量实验分离，不对混合变量图片作错误的因果暗示。
5. 自动识别只提供建议；用户必须能够手动整理 metadata 不完整的图片。
6. 上传结果图时通过 metadata 核对分支，任何不匹配都必须显式反馈。
7. 一个生成方案可以对应零张、一张或多张结果图。

## 4. 核心对象

### 4.1 结果图（Result）

从 NovelAI 或文件系统导入的实际图片，是不可修改的事实记录。

只读信息包括：

- 图片文件、文件哈希、原始文件名和导入时间。
- Base Prompt、Base Undesired Content。
- Character Prompt、Character Undesired Content 和位置。
- Prompt 顺序、权重和原始语法。
- Vibe 编码身份、Reference Strength、Information Extracted。
- Seed、Model、Sampler、Steps、CFG、尺寸及其他生成参数。
- 原始 metadata。

允许修改的用户注释包括：

- 展示名称、翻译、AI/手动分类。
- 备注、评分、收藏、颜色标签。
- 系列和对比实验成员关系。

### 4.2 分支方案（Branch Recipe）

用户第一次实际修改结果图的生成参数时，程序自动创建一个分支草稿。点击或聚焦字段不创建分支，只有值实际变化时才创建。

创建后界面必须显示：

- 分支名称和状态。
- 来源结果图或父分支。
- 相对来源发生变化的字段。
- 返回来源、放弃草稿和标记待生成操作。

分支状态：

- 草稿。
- 待生成。
- 等待结果图。
- 已有结果。
- 上传结果不匹配。

待生成方案一旦冻结，继续修改时应创建子分支，不能重写已经用于生成匹配的方案。

### 4.3 系列（Series）

系列是宽松的人工归档，例如“风笛花店”“夏装测试”“同一画师组合”。系列允许多个参数同时变化，不承担控制变量或因果解释。

系统可以根据文件夹、文件名、Base Prompt、角色和视觉相似度提供候选建议，但不应未经用户确认自动合并。

### 4.4 对比实验（Experiment）

对比实验用于明确控制变量。例如：

- 固定 Seed，只改变 Prompt/Tag。
- 固定 Base Prompt，只改变 Character Prompt。
- 固定 Prompt 和 Seed，只改变 Vibe。
- 固定 Prompt 和 Vibe，只改变 Seed。

实验应保存：

- 基准结果图。
- 成员结果图或待生成分支。
- 固定字段。
- 变化字段。
- 单变量或混合变量状态。

同一结果图可以加入多个对比实验。

## 5. 自动分组与对比镜头

不使用单一自动聚类规则，因为“与 A 同 Seed”“与 B 同 Base Prompt”具有不同含义，直接传递合并会形成错误的大组。

软件提供不同的对比镜头：

- 相同 Seed。
- 相同 Base Prompt。
- 相同完整 Prompt。
- 相同 Character Prompt。
- 相同 Vibe 来源或编码。
- 相同 Prompt/Vibe、不同 Seed。
- metadata 不完整。
- 可能的视觉相似图片。

用户可以从镜头结果中选择图片，建立系列或保存为对比实验。

## 6. 编辑与分支流程

```text
选择不可变结果图
    ↓ 修改任意生成参数
自动创建分支草稿
    ↓ 编辑 Prompt / Vibe / Seed / 其他参数
标记为“等待结果图”并冻结方案
    ↓ 在 NovelAI 中生成
上传一张或多张 PNG
    ↓ 解析并核对 metadata
完全匹配 → 绑定为该分支的结果图
部分不匹配 → 显示差异并建立子分支
```

如果分支使用随机 Seed，匹配时 Seed 是通配条件；结果图导入后记录实际 Seed。其他明确设置的字段仍需核对。

## 7. 结果上传与匹配

匹配依据包括：

- 规范化 Prompt 内容、结构、顺序和权重。
- Character Prompt、Undesired Content 和位置。
- Vibe 编码指纹、Strength、Information Extracted。
- Seed 或随机 Seed 规则。
- Model、Sampler、Steps、CFG 和尺寸。

匹配结果：

1. 唯一完全匹配：自动绑定。
2. 多个候选分支：让用户选择。
3. 部分不匹配：列出不同字段，保留原等待分支，并按实际 metadata 创建子分支。
4. 无候选：作为独立结果图导入，并推荐可能的系列或实验。

禁止用文件名作为唯一匹配依据。

## 8. 实验胶片

“版本胶片”建议改名为“实验胶片”或“系列胶片”，只显示当前系列或实验的成员。

卡片类型：

- 结果图：显示缩略图和锁定标识。
- 待生成分支：使用虚线占位和“等待结果图”状态。
- 多结果方案：显示结果数量。

卡片应显示相对基准的差异徽标，如 `Seed`、`Character`、`Vibe ×2`、`Base Prompt`、`3 项变化`。差异不能只通过颜色表达。

交互：

- 单击选择一个结果或分支。
- Ctrl/Cmd 多选 2–4 个成员进行对比。
- 基准图固定在首位。
- 支持水平滚动，但正文和状态文字不得小于 11px。
- 支持拖动调整实验中的展示顺序；拖动不改变来源关系。

## 9. 对比模式

对比模式占用中央工作区：

- 两张并排或最多 2 × 2 网格。
- 同步缩放、拖动和适配窗口。
- “视觉对比 / 参数差异”切换。
- 参数差异只显示变化字段，相同字段折叠。
- 明确显示“单变量对比”或“混合变量对比”。
- 混合变量实验提示“只能用于视觉筛选，不能判断单一参数影响”。

## 10. 数据完整性与边界情况

- metadata 不完整的图片显示明确状态，并允许手动归入系列。
- 后期处理、放大、裁剪图片可以与原结果建立“派生文件”关系，但不得冒充新的生成结果。
- 同一方案可以绑定多张候选结果图。
- 相同参数但不同图片文件不能自动视为重复文件。
- 删除系列或实验不删除原始结果图；删除图片必须单独确认。
- 所有分支和结果关系保留创建时间与来源，避免不可追踪的覆盖。

## 11. 推荐实施顺序

1. 拆分不可变结果图与可编辑用户注释。
2. 引入分支草稿、来源关系和状态机。
3. 将当前版本胶片迁移为实验胶片。
4. 增加手动系列和成员管理。
5. 增加 metadata 对比、智能候选和对比实验。
6. 增加待生成方案与上传结果匹配。
7. 最后实现多图同步对比和高级视觉相似建议。

## 12. 待确认决策

- 已采用“宽松系列 + 严格对比实验”两层结构；两者使用独立成员关系。
- 已确认：上传 PNG 不匹配时默认保留原分支，并自动建立“实际结果”子分支。
- 已确认：一个分支允许绑定多张结果图。
- 已采用多对多系列关系：一张图片可以同时属于多个系列；未来如需“主系列”，作为可选标记追加而不破坏现有成员关系。
